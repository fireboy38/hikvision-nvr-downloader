# 多设备并发下载功能说明

## 功能概述

海康NVR批量录像下载工具现已支持**多设备并发下载**，可同时从多台NVR下载录像，并优化了界面布局。

## 主要改进

### 1. 多设备支持 ✅

- **通道分组显示**：按设备分组显示通道，树形结构清晰明了
- **并发下载**：支持同时从多台NVR下载录像
- **设备独立管理**：每个设备独立连接，互不干扰
- **智能任务分配**：自动根据通道所属设备创建下载任务

### 2. 界面优化 ✅

- **设置对话框**：将打包和线程设置合并到弹出对话框
- **节省空间**：左侧面板更简洁，只保留核心功能
- **快速访问**：工具栏添加"⚙️ 设置"按钮
- **提示友好**：每个设置项都有详细的说明

## 使用方法

### 添加多台设备

1. 点击"➕ 添加设备"添加第一台NVR
2. 重复步骤添加第二台、第三台...NVR
3. 选择设备后点击"🔌 连接设备"获取通道列表

### 选择通道下载

1. 在左侧"通道选择"树形列表中查看所有设备的通道
2. 展开/折叠设备节点查看通道
3. 勾选需要下载的通道（可以跨设备选择）
4. 设置时间范围
5. 点击"▶ 开始下载"

### 调整设置

1. 点击工具栏"⚙️ 设置"按钮
2. 在弹出的对话框中调整：
   - **并发线程数**：1-10线程，建议6
   - **打包时长**：0-720分钟，0表示不打包
   - **删除原始文件**：打包后是否删除原始文件
3. 点击"确定"保存设置

## 技术架构

### 多设备下载流程

```
用户选择通道（跨多台设备）
    ↓
按设备分组创建任务
    ↓
设备A: 任务1, 任务2, 任务3
设备B: 任务4, 任务5
    ↓
提交到DownloadManager
    ↓
多线程并发执行
    ↓
每个任务调用对应的Java进程
    ↓
Java进程独立登录对应的NVR
    ↓
并发下载
```

### 关键设计

1. **设备分组**：
   - 通道树按设备分组显示
   - 每个设备节点显示通道数
   - 支持展开/折叠

2. **任务创建**：
   - 按设备分组创建下载任务
   - 每个任务包含设备配置
   - 下载管理器自动路由到对应设备

3. **并发控制**：
   - 全局线程池管理所有下载任务
   - 不同设备的任务可以并发执行
   - 同一设备的多个任务也并发执行

## 性能优化

### 多设备优势

- **带宽利用**：多台设备同时下载，充分利用网络带宽
- **效率提升**：2台设备 = 2倍速度（理论上）
- **任务均衡**：自动分配任务到不同设备

### 推荐配置

| 场景 | 线程数 | 说明 |
|------|--------|------|
| 单台NVR | 6线程 | 标准配置 |
| 2台NVR | 8-10线程 | 充分利用带宽 |
| 3台以上NVR | 10线程 | 最大并发 |

## 注意事项

### 设备连接

- **必须先连接**：下载前必须先连接对应的设备
- **独立管理**：每个设备需要单独连接
- **连接状态**：通道树中只显示已连接设备的通道

### 通道选择

- **跨设备选择**：可以同时选择多台设备的通道
- **批量操作**：使用"全选"/"取消全选"快速选择
- **设备节点**：设备节点本身不可选中

### 下载性能

- **网络带宽**：确保网络带宽足够支持多设备并发
- **NVR性能**：老旧NVR建议减少并发任务
- **磁盘空间**：确保有足够磁盘空间存储录像

## 故障排查

### 问题：部分设备下载失败

**可能原因**：
- 对应设备未连接
- 设备配置错误
- 网络不通

**解决方案**：
1. 检查设备是否已连接
2. 测试设备连接状态
3. 检查网络配置

### 问题：所有下载速度都慢

**可能原因**：
- 网络带宽不足
- 线程数过多
- NVR性能瓶颈

**解决方案**：
1. 检查网络带宽使用情况
2. 减少并发线程数
3. 分批下载通道

### 问题：找不到通道

**可能原因**：
- 设备未连接
- 通道列表未刷新

**解决方案**：
1. 选择设备后点击"连接设备"
2. 如果仍找不到，点击"刷新通道"
