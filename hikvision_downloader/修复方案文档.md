# 海康NVR下载器 - 问题修复说明

## 问题描述

用户反馈下载3.2GB的视频时，文件被SDK自动分割成多个1GB左右的片段文件：
- 文件1：~1GB
- 文件2：~1GB
- 文件3：~200MB

用户希望下载的录像不要被分割，而是合并为一个完整的文件。

## 问题原因分析

海康SDK的 `NET_DVR_GetFileByTime` 接口在下载大文件时，会根据内部配置自动分割文件，通常每个文件约1GB大小。

**这不是bug，而是SDK的设计**：
- 便于网络传输断点续传
- 降低单个文件损坏的风险
- 兼容旧的FAT32文件系统（最大2GB）

## 解决方案

有两种方式解决这个问题：

### 方案1：自动合并下载后的文件（推荐）

下载完成后，自动将同一通道的多个片段文件按时间顺序合并为一个完整文件。

**实现步骤**：
1. 下载完成后，检查是否有同一通道的多个文件
2. 使用FFmpeg按文件名排序后合并
3. 删除原始片段文件（可选）

**优点**：
- 无需修改SDK调用逻辑
- 保持现有下载流程不变
- 稳定可靠

**缺点**：
- 需要临时存储片段文件
- 合并过程需要额外时间

### 方案2：修改SDK配置（不推荐）

尝试通过SDK配置禁用文件分割，但：
- SDK未公开相关配置接口
- 可能导致下载失败或数据损坏
- 不建议尝试

## 推荐实现：方案1 - 自动合并

### 实现位置

在 `core/hiksdk_downloader.py` 的 `download_by_time` 函数中，下载完成后添加合并逻辑。

### 代码实现

```python
def download_by_time(
    channel: int,
    start_time: datetime,
    end_time: datetime,
    save_path: str,
    progress_callback: Optional[Callable[[int], None]] = None,
    timeout: int = 7200
) -> Tuple[bool, str]:
    """使用海康SDK按时间下载录像"""

    # ... 原有的下载逻辑 ...

    # 下载完成后，检查是否被分割成多个文件
    base_dir = os.path.dirname(save_path)
    base_name = os.path.splitext(os.path.basename(save_path))[0]
    ext = os.path.splitext(save_path)[1]

    # 查找所有同时间段的文件（SDK分割的文件）
    pattern = f"{base_name}*{ext}"
    related_files = sorted(glob.glob(os.path.join(base_dir, pattern)))

    # 如果找到多个文件（说明被分割了），进行合并
    if len(related_files) > 1:
        print(f"[HikSDK] 检测到 {len(related_files)} 个文件，开始合并...")

        # 生成合并后的文件名（去掉可能的分割标识）
        merged_path = save_path  # 使用原始保存路径

        # 创建concat列表
        concat_file = os.path.join(base_dir, f"concat_{int(time.time())}.txt")
        with open(concat_file, 'w', encoding='utf-8') as f:
            for file in related_files:
                abs_path = os.path.abspath(file).replace('\\', '/')
                f.write(f"file '{abs_path}'\n")

        # 使用FFmpeg合并
        cmd = [
            r"C:\tools\ffmpeg\bin\ffmpeg.exe",
            '-y',
            '-f', 'concat',
            '-safe', '0',
            '-i', concat_file,
            '-c', 'copy',
            merged_path
        ]

        result = subprocess.run(
            cmd,
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE,
            timeout=3600
        )

        # 删除临时文件
        try:
            os.remove(concat_file)
        except:
            pass

        if result.returncode == 0:
            # 合并成功，删除原始片段文件
            print(f"[HikSDK] 合并成功: {merged_path}")
            for file in related_files:
                if file != merged_path and os.path.exists(file):
                    os.remove(file)
                    print(f"[HikSDK] 已删除片段: {os.path.basename(file)}")
            return True, "下载并合并成功"
        else:
            print(f"[HikSDK] 合并失败: {result.stderr}")
            # 合并失败，保留原始文件
            return True, "下载完成（合并失败，保留片段）"

    return True, "下载成功"
```

## 用户操作指南

### 方案1（推荐）：等待自动合并

1. 勾选"自动合并分割文件"（默认勾选）
2. 正常下载录像
3. 下载完成后，程序会自动：
   - 检测到多个片段文件
   - 使用FFmpeg合并为一个完整文件
   - 删除原始片段文件
4. 下载完成后得到单个完整录像文件

### 方案2：手动合并

1. 如果未勾选自动合并，下载会保留所有片段文件
2. 下载完成后，点击"📦 打包录像"按钮
3. 程序会按时长分组合并文件

## 预期效果

**下载前**：
- 请求下载：2026-03-24 20:00 - 23:00（3小时录像）

**下载后（自动合并）**：
- 得到单个文件：`NVR_正大门停车场1_20260324_200000_230000.mp4`
- 文件大小：~3.2GB
- 时长：3小时（180分钟）

**如果不合并**：
- 文件1：`NVR_正大门停车场1_20260324_200000_230000.mp4_1` (~1GB)
- 文件2：`NVR_正大门停车场1_20260324_200000_230000.mp4_2` (~1GB)
- 文件3：`NVR_正大门停车场1_20260324_200000_230000.mp4_3` (~200MB)

## 注意事项

1. **临时文件**：合并过程中需要临时存储片段文件，需要足够的磁盘空间
2. **合并时间**：合并大文件需要时间，3.2GB的文件可能需要几分钟
3. **FFmpeg依赖**：需要确保FFmpeg已安装并配置正确路径
4. **网络稳定性**：下载大文件时请保持网络稳定，避免多次分段

2. 确认合并后的文件能正常播放
3. 再下载大文件（1小时+）测试性能
