# 多线程下载功能说明

## 功能概述

海康NVR批量录像下载工具现已支持多线程并发下载，可显著提升下载速度。

## 主要特性

- **可配置并发线程数**：1-10线程可选，默认6线程
- **独立下载进程**：每个线程调用独立的Java进程，避免SDK冲突
- **实时进度监控**：每个任务的下载进度独立显示
- **智能队列管理**：任务队列自动分配给空闲线程

## 使用方法

### GUI界面

1. 启动程序：
   ```bash
   cd hikvision_downloader
   python main.py
   ```

2. 在左侧面板找到"下载线程设置"分组

3. 设置并发线程数：
   - 范围：1-10
   - 建议：6线程（平衡速度与稳定性）
   - 提示：过多线程可能导致NVR响应变慢

4. 选择多个通道进行下载

5. 点击"▶ 开始下载"

## 技术架构

### 多线程实现

```
DownloadManager (主进程)
    ├── 工作线程1 → Java进程1 → NVR
    ├── 工作线程2 → Java进程2 → NVR
    ├── 工作线程3 → Java进程3 → NVR
    ├── ...
    └── 工作线程N → Java进程N → NVR
```

### 关键设计

1. **每个工作线程独立**：
   - 从任务队列获取任务
   - 启动独立的Java进程
   - Java进程独立初始化SDK
   - Java进程独立登录NVR

2. **无设备登录冲突**：
   - 每个Java进程使用独立的用户ID
   - SDK支持多用户同时登录
   - NVR限制通常在10个并发连接以内

3. **进度监控**：
   - 每个Java进程输出进度信息
   - Python主进程解析并更新UI
   - 支持实时进度显示

## 性能对比

### 单线程 vs 多线程

假设下载3个通道，每个通道100MB录像：

| 模式 | 理论耗时 | 实际速度提升 |
|------|---------|-------------|
| 1线程 | 300秒 | 基准 |
| 6线程 | 50秒 | 6倍 |
| 10线程 | 30秒 | 10倍 |

*注：实际速度受网络带宽和NVR性能限制*

## 注意事项

### 推荐配置

- **一般使用**：6线程
- **高速网络**：8-10线程
- **低速网络**：3-4线程
- **老旧NVR**：2-3线程

### 性能瓶颈

1. **网络带宽**：
   - 千兆网络：建议8-10线程
   - 百兆网络：建议3-6线程

2. **NVR性能**：
   - 高端NVR（128路）：支持10线程
   - 中端NVR（64路）：推荐6线程
   - 低端NVR（32路）：推荐3-4线程

3. **磁盘I/O**：
   - SSD：无明显瓶颈
   - HDD：建议不超过6线程

### 潜在问题

1. **NVR响应变慢**：
   - 症状：连接超时、查询缓慢
   - 解决：减少线程数

2. **下载失败增加**：
   - 症状：部分任务失败
   - 解决：减少线程数或重试失败任务

3. **网络拥堵**：
   - 症状：所有下载速度都变慢
   - 解决：减少线程数或检查网络带宽

## 故障排查

### 问题：下载失败率高

**可能原因**：
- 线程数过多
- NVR负载过高
- 网络带宽不足

**解决方案**：
1. 减少线程数到3-4
2. 增加下载超时时间
3. 分批下载通道

### 问题：NVR连接超时

**可能原因**：
- NVR同时连接数达到上限
- NVR资源耗尽

**解决方案**：
1. 立即停止下载
2. 等待1-2分钟让NVR恢复
3. 重新开始时使用更少线程数

### 问题：下载速度慢

**可能原因**：
- 线程数过少
- 网络带宽不足
- NVR性能限制

**解决方案**：
1. 增加线程数（不超过10）
2. 检查网络带宽
3. 检查NVR CPU使用率

## 代码结构

### 核心文件

- `gui/main_window.py` - GUI界面，包含线程数设置
- `core/downloader.py` - 下载管理器，支持多线程
- `core/java_downloader.py` - Java下载器封装
- `test_multithread.py` - 多线程测试脚本

### 关键类

- `DownloadManager` - 下载管理器
  - `max_concurrent` - 最大并发数
  - `_worker()` - 工作线程函数
  - `start()` - 启动多线程下载

- `DownloadTask` - 下载任务
  - 每个任务对应一个通道的一个时间段
